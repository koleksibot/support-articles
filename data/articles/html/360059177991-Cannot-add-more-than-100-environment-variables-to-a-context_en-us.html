The maximum number of variables that are configurable for one context is limited to 100 variables. Contexts created prior to this limit being introduced will still be allowed to have more than 100 variables, but new ones will not be able to be added until the total number is lower than 100.

As a workaround, you can create additional contexts for new variables, and then include multiple contexts in your builds.

Also, it is now possible to modify the values of variables directly using the context endpoints of our V2 API
