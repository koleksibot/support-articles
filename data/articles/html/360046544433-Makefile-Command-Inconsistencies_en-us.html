Occasionally, Makefile commands won't behave as expected within a Docker image - for example, creating multiple directories with mkdir -p build/{assets,bin} will produce a directory named {assets,bin}/Â rather than two directories (assets/ and bin/). This is typically because make does not know which shell to use, defaulting to one that does not handle the command parameters as expected. Your Makefile should have the following line added to ensure bash is used:

SHELL := /bin/bash
