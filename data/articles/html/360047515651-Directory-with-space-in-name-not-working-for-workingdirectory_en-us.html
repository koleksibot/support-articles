When setting the working_directory in a build, it sets the CIRCLE_WORKING_DIRECTORY environment variable to that value, so if the directory has a space like "01 Test" it would set it to:

CIRCLE_WORKING_DIRECTORY=01 Test

So when it goes to create the directory it calls mkdir 01 Test -- which would result in two directories being created (01 and Test). With this in mind, you can protect the space with single quotes like this:

working_directory: "01' 'Test"

That should result in the variable being set to CIRCLE_WORKING_DIRECTORY=01' 'Test which should properly account for the space.
